# **Планирование**
## **Позитивные кейсы**
### *1. Успешная оплата дебетовой картой*
**Шаги:**
1. Нажать кнопку "Купить" на главной странице
2. В появившееся форме "Оплата по карте" нажать на поле ввода "Номер карты", ввести валидную карту (4444 4444 4444 4441)
3. Нажать на поле ввода "Месяц", ввести валидный месяц (12)
4. Нажать на поле ввода "Год", ввести валидный год (25)
5. Нажать на поле ввода "Владелец", ввести валидное имя (Иванов Иван)
6. Нажать на поле ввода "CVC", ввести валидный CVC (123)
7. Нажать кнопку "Продолжить"

**Ожидаемый результат:**
- На экране появляется уведомление: "Успешно! Операция одобрена Банком"

## Используемые инструменты:
- Java
- Gradle
- Selenide - позволяет писать более читаемые и короткие тесты
- JUnit5 - cтандартный тестовый фреймворк для Java
- JDBC - инструмент для работы с базой данных
- Faker - позволяет генерировать реалистичные фиктивные данные для заполнения форм
- Allure - генерирует красивые, подробные отчеты о тестах
- DevTools - позволяет находить уникальные локаторы
- DBeaver - для ручной проверки и управления базами данных
- Docker - позволяет упаковывать СУБД и эмулятор банковских сервисов в изолированные контейнеры
- GitHub - позволяет отслеживать все изменения в коде, возвращаться к предыдущим версиям

## Возможные риски:
- Изменения в структуре элементов могут привести к неработоспособности локаторов, используемых в тестах
- Тесты могут по-разному вести себя в разных браузерах
- Проблемы с СУБД, контейнеры с базами данных могут не стартовать, иметь проблемы с подключением или очисткой данных, если данные из предыдущего теста не удалены или подготовлены неправильно
- Зависимость от портов, если порты 8080, 9999, 3306, 5432 уже заняты другими процессами на машине, Docker-контейнеры могут не запуститься
- Отсутствие ресурсов для поддержки и развития автоматизации
